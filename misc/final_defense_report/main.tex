% --- Abstract ---
\chapter*{ABSTRACT}
\addcontentsline{toc}{chapter}{ABSTRACT}

The Blood Donor Information Management System (BDIMS) is a comprehensive web-based application designed to modernize and streamline blood donation management for healthcare institutions in Nepal. The system addresses critical challenges in donor information management, blood inventory tracking, and emergency response coordination through a centralized digital platform.
\\\\
Built using Django framework and PostgreSQL database, BDIMS provides role-based access for both administrators and donors. The system enables administrators to manage comprehensive donor databases, track real-time blood inventory across multiple centers, handle emergency blood requests, monitor donor eligibility with automated 56-day rule calculations, and generate detailed analytical reports. Donors can maintain detailed personal and medical profiles, track health metrics over time, view complete donation histories, update location information with interactive map integration, and respond to emergency blood requests.
\\\\
The system incorporates advanced features including interactive Leaflet.js-based maps for location tracking, automated eligibility verification, multi-channel notification system, health metrics monitoring, blood compatibility checking, and comprehensive reporting capabilities. Security is ensured through role-based authentication, password hashing, and session management.
\\\\
BDIMS successfully replaces traditional paper-based systems with an efficient, scalable digital solution that improves coordination between blood banks, hospitals, and donors. The system has been thoroughly tested and optimized for performance, achieving 100\% functionality for all core features including donor management, inventory tracking, emergency request handling, and analytical reporting.
