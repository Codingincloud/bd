\chapter*{ABSTRACT}
\addcontentsline{toc}{chapter}{Abstract}

Our project Blood Donor Information Management System is a comprehensive web-based application designed to centralize and manage detailed blood donor information for healthcare institutions. The system focuses on organizing comprehensive donor profiles, tracking health metrics, managing donation histories, and providing advanced search capabilities for donor information. Built on a centralized PostgreSQL database, it ensures secure storage and efficient retrieval of donor data with role-based authentication for administrators and donors.

This information management platform enables administrators to access detailed donor databases, track comprehensive health metrics, monitor eligibility status with automated calculations, manage donor profiles with medical information, and generate analytical reports. Donors can maintain detailed personal and medical profiles, track their health metrics over time, view complete donation histories, and update their information. The Django-based system provides efficient donor information organization and management capabilities for modern healthcare institutions.

\vspace{1em}
\noindent\textbf{Keywords:} \textit{Blood Donor Information Management, Django Framework, PostgreSQL, Emergency Requests, Donor Eligibility, Healthcare Analytics}

\chapter{INTRODUCTION}

\section{BACKGROUND}
Healthcare institutions struggle with managing comprehensive donor information including medical records, health metrics, donation history, and eligibility tracking. Our Blood Donor Information Management System provides a centralized platform built with Django framework and PostgreSQL database to efficiently organize, store, and manage all donor-related information.

The system focuses on comprehensive donor information management with detailed profile tracking, health metrics monitoring, automated eligibility calculations, and advanced search capabilities. Administrators can access complete donor databases, track health metrics, manage blood inventory, and generate detailed reports. Donors maintain detailed profiles with medical information, view comprehensive donation history, and receive personalized health tracking.

\section{PROBLEM STATEMENT}
Healthcare institutions face significant challenges in managing comprehensive donor information including scattered medical records, manual health metrics tracking, inefficient donor search systems, and lack of centralized donor databases. Traditional paper-based systems cannot handle complex donor information requirements such as detailed health metrics, blood compatibility tracking, emergency contact management, and automated eligibility calculations. This creates inefficiencies in donor information retrieval, health monitoring, and emergency response coordination.

\section{OBJECTIVES}
The primary objective is to create a centralized donor information management platform that organizes, stores, and manages comprehensive donor data efficiently. Specific goals include developing detailed donor profile management, implementing health metrics tracking, creating advanced search and filtering capabilities, establishing automated eligibility calculations, and providing comprehensive donor information analytics for healthcare decision-making.

\chapter{METHODOLOGY}


\section{ALGORITHM}
\begin{enumerate}
    \item \textbf{System Initialization:} Initialize Django application, establish PostgreSQL database connection, load system configurations

    \item \textbf{User Access Control:} Display home page with role-based authentication options (Donor/Admin)

    \item \textbf{Authentication Process:}
    \begin{enumerate}
        \item Collect username, password, and role selection
        \item Validate credentials against User model in PostgreSQL database
        \item Verify role permissions (is\_staff for admin, donor profile existence for donors)
        \item Create secure session with Django authentication system
        \item Redirect to appropriate dashboard based on role validation
        \item Log authentication attempts and handle database connection errors
    \end{enumerate}
    \item \textbf{Donor Information Management:}
    \begin{enumerate}
        \item Retrieve donor profile from Donor model with One\-To\-One\-Field relationship to User
        \item Calculate eligibility status using can\_donate() method with 56-day rule validation
        \item Display comprehensive donor information (blood\_group, medical\_conditions, emergency\_contact)
        \item Show donation history from DonationHistory model ordered by donation\_date
        \item Present health metrics from HealthMetrics model (weight, blood\_pressure, heart\_rate)
        \item Enable profile updates with form validation and database transaction handling
    \end{enumerate}

    \item \textbf{Emergency Request Processing:}
    \begin{enumerate}
        \item Query EmergencyRequest model for active requests matching donor's blood\_group
        \item Filter by urgency\_level and required\_by datetime for priority ordering
        \item Check blood compatibility using compatible\_blood\_groups property
        \item Send notifications to eligible donors using NotificationService
        \item Update request status (active/fulfilled/expired) based on response
        \item Track emergency contact preferences with allow\_emergency\_contact flag
    \end{enumerate}
    \item \textbf{Blood Inventory Management:}
    \begin{enumerate}
        \item Access BloodInventory model with blood\_group as unique identifier
        \item Calculate inventory status (critical/low/medium/good) based on units\_available
        \item Update inventory levels with last\_updated time\-stamp and updated\_by user tracking
        \item Monitor blood expiry using 35-day calculation from donation\_date
        \item Generate inventory alerts for low stock levels by blood group
        \item Maintain inventory notes for tracking changes and updates
    \end{enumerate}

    \item \textbf{Admin Dashboard Analytics:}
    \begin{enumerate}
        \item Aggregate statistics: total\_donors, total\_donations, pending\_requests, active\_emergencies
        \item Calculate daily metrics: today\_donations, today\_requests from filtered query\-sets
        \item Generate monthly statistics: month\_donations, month\_new\_donors using date filtering
        \item Create blood\_group\_stats with donor count and donation count per blood type
        \item Display recent activities: recent\_donations, recent\_requests, pending\_approvals
        \item Show urgent\_emergencies filtered by urgency\_level and required\_by datetime
    \end{enumerate}
    \item \textbf{Donor Search and Management:}
    \begin{enumerate}
        \item Implement advanced search using DonorAdmin with multiple search\_fields
        \item Filter donors by blood\_group, gender, is\_eligible, city, country, created\_at
        \item Display donor information: name, age, phone\_number, total\_donations, last\_donation\_date
        \item Enable bulk operations and individual donor profile management
        \item Track donor eligibility with automated next\_eligible\_date calculation
        \item Manage emergency contacts and medical condition updates
    \end{enumerate}

    \item \textbf{Notification System:}
    \begin{enumerate}
        \item Create System\-Notification for system-wide announce\-ments with target\_audience filtering
        \item Generate UserNotification for individual users with notification\_type classification
        \item Prevent duplicate notifications using timestamp-based filtering (one hour window)
        \item Handle donation completion notifications with next eligible date calculation
        \item Process emergency request notifications to compatible donors
        \item Track notification read status with SystemNotificationRead model
    \end{enumerate}
    \item \textbf{Health Metrics Tracking:}
    \begin{enumerate}
        \item Record health data in Health\-Metrics model: weight, blood\_pressure\_systolic/\allowbreak diastolic, resting\_heart\_rate
        \item Timestamp health records with recorded\_at for historical tracking
        \item Display health trends and medical reports for donor monitoring
        \item Validate health metrics against donation eligibility criteria
        \item Enable health data updates with form validation and error handling
        \item Generate health reports for medical assessment and donor screening
    \end{enumerate}

    \item \textbf{Database Transaction Management:} Execute all operations with Django ORM transactions, implement error handling with try-catch blocks, maintain data integrity with foreign key constraints

    \item \textbf{Session Security:} Maintain secure user sessions with Django authentication, implement logout with session cleanup, log security events for audit trails
\end{enumerate}

        \item Manage Blood Centers
        \item Track Donors

        \item If "Generate Reports" (Admin) - Create analytics and statistical reports.
